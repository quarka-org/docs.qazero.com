# QA とは

QA ZERO は、もっと大きな何かの1つのエディションです。その大きな何か
— **QA**、エディションが育つ幹 — を理解することが、この API がなぜ
この形をしているのかを理解する一番の近道です。

## QA は Behavioral Intelligence Architecture です

従来の Web 分析はメトリクスを報告します: ページビュー、セッション、
直帰率、コンバージョン。QA の目的は違います。

> QA は、人間が情報とどうやり取りするのかを AI が理解できるように
> するために存在します。

人間が読むためのレポートを作るのではなく、QA は **AI が推論に使える
構造化された行動知識** を組み立てます。だからこそ QA は、その核心に
おいては分析プラットフォームではありません。Behavioral Intelligence
Architecture（行動インテリジェンスアーキテクチャ）です。

生の行動イベントは、それ単体では何の意味も持ちません。QA は観測可能な
行動を構造化された知識へと変換します。そのゴールはデータを集めること
ではなく、AI が人間の意図を理解できるようにすることです。

```
Human → Behavior → Structured Events → Semantic Labels
     → Knowledge → Reasoning → Recommendation → Better Human Experience
```

可視化はあり得る出力の1つにすぎません。理解こそが本当の目的です。

## すべての行動イベントに共通する3つの原則

- **Observable（観測可能）** — 客観的に測定できる（スクロール、
  クリック、ホバー、読了時間、再訪問、ナビゲーション）。
- **Contextual（文脈依存）** — 同じ行動でも、サイトが違えば意味が
  違います。3分の読了は、ドキュメントでは *熱心な学習*、EC ページ
  では *ためらい*、ニュースでは *普通* を意味します。したがって
  文脈はデータの周りにあるメタデータではなく、データの一部です。
- **Semantic（意味的）** — 意味は統計的推論だけに頼るべきでは
  ありません。QA は明示的なセマンティックラベル（*じっくり読んで
  いる*、*製品を比較している*、*ためらっている*、*タスク完了*）を
  サポートし、それらが AI にとって高品質なシグナルになります。

## なぜラベルが正確なのか: 集合知

ラベルは正確であって初めて価値があります。そして正確さこそが難しい
部分です — なぜなら、同じ行動の値でも Web サイトのタイプによって
意味が違うからです。ですからラベルはグローバルに適用するのではなく、
**文脈（サイトタイプ）ごとにセグメント分け** されなければなりません。

QA の答えは集合知です: WordPress サイトの分散ネットワーク全体で、
人間と AI が協働し、各サイトセグメントに固有の推論精度を高めます。
この仕組みは特許 **JP 7011367** の対象です。データは各サイト自身の
サーバーに留まる — Cookie レス、匿名化、中央クラウドなし — ため、
QA は中央集権的な分析への、分散型・ファーストパーティな対抗軸です:
各サイトは自律的に動きながら、そのすべてが共有された理解を改善します。

## これが QA ZERO の API をどう形づくるか

この開発者マニュアルのすべては、上記の考え方から導かれています:

- **SQL ではなく QAL。** [QAL](./concepts/what-is-qal.md) は小さな
  宣言型言語で、AI アシスタントが最初の一発で正しいクエリを組み立て
  られ、かつ不正なクエリが裏のストアを傷つけられないように形づくられ
  ています。
- **マテリアルは単なる行ではなく意味を運ぶ。** クエリ対象の
  [マテリアル](./api/materials/)は、生のヒットだけでなく行動の文脈
  （深読み、いらだちクリック、エンゲージメントの深さ）をすでに
  エンコードしています。
- **仕様は機械可読でファーストクラス。** [AI 向けの仕様](./for-ai/)は
  人間向けドキュメントの隣で配布されるので、LLM、MCP サーバー、
  エージェントが直接それを消費できます。

## QA は GA4 を置き換えない

QA は GA4 と **併用** することを意図しています。GA4 は *何が起きたか*
を測定します。QA は *それが何を意味したか* と *次に何をすべきか* を
足すので、人々は数字を読み解く代わりに自分のデータをもとに行動でき
ます。本当の目的は創造性のループです: 人々がようやく自分のデータの
力を使えるようになると、メトリクスの解釈に費やすエネルギーが減り、
自分自身の創造性を表現するエネルギーが増えます — そしてその創造性は、
行動として観測され、インテリジェンスをより良くするためにフィード
バックされます。
