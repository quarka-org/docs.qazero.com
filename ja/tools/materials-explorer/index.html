Materials Explorer（静的版） — 全マテリアル × フィールド一覧。 インタラクティブ表示には JavaScript が必要です（JS対応ブラウザではこの一覧は非表示になり、上の対話UIが表示されます）。 AI・MCP・エージェントから使う場合は、機械可読の正本 /wp-json/qa-platform/guide または for-ai/materials.yaml を参照してください。

allpv — 1行 = 1ページビュー

全ページビューの生データ。熟読・離脱位置・イライラクリック等の行動シグナルも同じ行に持つ。

tracking_id "all"（全サイト横断集計）: 対応

キー & ID（キー）

JOIN と絞り込みに使う識別子。フィルタは通常 url / device_type 等の解決済みカラムで行う。

  • pv_id [uint32] [join] — ページビュー識別。click_event 等との結合キー。
  • session_id [uint64] [join] — セッション単位の絞り込み。列DBバイナリスキャンで高速。
  • reader_id [uint32] [join] — ユーザー単位の絞り込み。url 等と組み合わせて使う。
  • page_id [uint32] [join] — ページ単位。通常は url で絞り、page_id は JOIN キー。
  • device_id [uint8] — 端末ID。1=PC, 2=SP, 3=tablet。ラベルは device_type。
  • source_id [uint16] — 流入元ID。ラベルは utm_source。
  • medium_id [uint8] — メディアID。ラベルは utm_medium。
  • campaign_id [uint16] — キャンペーンID。ラベルは utm_campaign。
  • content_id [uint16] — コンテンツID。ラベルは utm_content。
  • version_id [uint16] — ページ版ID。A/B・バージョン比較に使用。
  • prev_page_id [uint32] [since 2026-04-17] — 前ページID。0 でランディング抽出。
  • next_page_id [uint32] [since 2026-04-17] — 次ページID。0 で離脱PV抽出。

セッションの基本（シグナル）

閲覧の基礎メトリクス。範囲フィルタ可（browse_sec gte:30 等）。

  • access_time [uint32] — UNIX時刻。通常は期間パラメータで自動処理。時間帯絞り込みも可。
  • pv [uint16] — セッション内のPV番号（1=ランディング）。
  • speed_msec [uint16] — 表示速度(ms)。「3秒以上かかったPV」等も抽出可。
  • browse_sec [uint16] — 閲覧時間(秒)。
  • is_last [uint8] — セッション最終PV(0/1)。離脱ページ抽出に。
  • is_newuser [uint8] — 新規ユーザー(0/1)。

読了の深さ（シグナル）

PV単位のスクロール到達率・熟読・停止位置。

  • depth_position [uint8] [✦] — 最深到達率(0-100%)。gte:80 で下部まで読了、lte:20 で冒頭離脱。
  • deep_read [uint8] [✦] — 熟読フラグ。3秒以上滞在の区間が5箇所以上で1。
  • stop_max_sec [uint16] — 最長停止時間(秒)。最も長く留まった箇所。
  • stop_max_pos [uint8] — 最長停止位置(0-100%)。stop_max_sec とセット。
  • exit_pos [uint8] [✦] — 離脱位置(0-100%)。lte:30 でファーストビュー内離脱。

フラストレーション（シグナル）

連打・デッドクリック・フォーム送信。UI不満のシグナル。

  • irritation_click_count [uint8] [✦] — 3秒以内に5回以上の連打バースト数。gte:1 でUI不満。
  • dead_click_image_count [uint8] [✦] — リンクの無い画像へのクリック数。gte:1 で機会損失。
  • is_submit [uint8] — フォーム送信(0/1)。フォームCVR算出に。

探索行動（シグナル）

見直し・飛ばし読み・迷いのマウス挙動。ナビゲーションの質を測る。

  • scroll_back_count [uint8] [✦] — 3秒以内に1000px以上上へ戻った回数=見直し。
  • content_skip_count [uint8] [✦] — 1000px以上下へ飛ばした回数=読まれていない。
  • exploration_count [uint8] [✦] — マウスが横に往復した回数=クリック対象を探して迷子。

ビューポート（シグナル）

実表示領域サイズ。レスポンシブ帯別のセグメントに。

  • window_inner_width [uint16] — innerWidth(px)。between:768,1199 でタブレット帯、gte:1200 でPC帯。
  • window_inner_height [uint16] — innerHeight(px)。

ページ遷移（シグナル）

前後どのページを見ていたか。導線分析に。

  • prev_url [since 2026-04-17] — 前ページURL。ランディングPVは null。
  • prev_title [since 2026-04-17] — 前ページタイトル。
  • next_url [since 2026-04-17] — 次ページURL。離脱PVは null。
  • next_title [since 2026-04-17] — 次ページタイトル。

流入元 & リファラー（シグナル）

どこから来たか。

  • url — ページURL。完全一致は url_hash で高速。
  • title — ページタイトル。url で絞って keep する方が効率的。
  • source_domain — 流入元ドメイン。インデックスあり・高速。
  • referrer — リファラー。
  • utm_source — 流入元。
  • utm_medium — メディア。ユニークインデックス・高速。
  • utm_campaign — キャンペーン。ユニークインデックス・高速。
  • utm_content — コンテンツ。
  • utm_term — 検索語(utm)。

デバイス & ユーザー（シグナル）

端末・OS・言語・国。

  • device_type — PC / SP / tablet の3値（正確な大小文字で）。
  • os — OS(UAos)。インデックスあり・高速。
  • browser — ブラウザ(UAbrowser)。
  • language — 言語。
  • country_code — 国コード(約200種)。インデックスあり・高速。
  • ua — UA合成。直接フィルタ非推奨、os/browser を個別に。
  • original_id — 元ID。インデックスあり・高速。

  • page_type — ページタイプ・ビットフラグ。通常は is_* で絞る方が効率的。
  • page_fetch_status — HTML取得状態。1=成功, -1=失敗。
  • is_article — 記事(0/1)。
  • is_product — 商品(0/1)。
  • is_list — 一覧(0/1)。
  • is_form — フォーム(0/1)。
  • is_trust_info — 信頼性情報(0/1)。
  • is_faq — FAQ(0/1)。
  • is_landing — ランディング(0/1)。
  • is_search — 検索結果(0/1)。
  • is_account — アカウント(0/1)。
  • is_cart — カート(0/1)。
  • is_checkout — 決済(0/1)。
  • is_confirm — 確認(0/1)。
  • is_thanks — サンクス(0/1)。
  • is_top_page — トップ(0/1)。
  • is_event — イベント(0/1)。
  • is_recipe — レシピ(0/1)。
  • is_job — 求人(0/1)。
  • is_video — 動画(0/1)。
  • is_howto — ハウツー(0/1)。
  • is_qa_forum — Q&Aフォーラム(0/1)。

ゴール達成（シグナル）

  • is_goal_1 [uint8] — ゴール1達成(0/1)。
  • is_goal_2 [uint8] — ゴール2達成(0/1)。
  • is_goal_3 [uint8] — ゴール3達成(0/1)。

  • is_goal_7 [uint8] — ゴール7達成(0/1)。
  • is_goal_8 [uint8] — ゴール8達成(0/1)。
  • is_goal_9 [uint8] — ゴール9達成(0/1)。
  • is_goal_10 [uint8] — ゴール10達成(0/1)。

click_event — 1行 = 1クリック

全クリックを自動保存している生データ。セレクタや座標なども保有し、生PVログとJOIN可能。

tracking_id "all"（全サイト横断集計）: 対応

キー & ID（キー）

allpv と結合するキー。

  • pv_id [uint32] [join] — どのPV上のクリックか。allpv と結合。
  • session_id [uint64] [join] — セッション単位。
  • page_id [uint32] [join] — ページ単位。

クリックの中身（シグナル）

何が・いつ押されたか。event_sec で「読んでから押したか」が分かる。

  • event_sec [uint16] [✦] — ページ表示からクリックまでの秒数。
  • element_text [uint16] — クリックされた要素のテキスト。
  • selector [uint32] — CSSセレクタ。
  • to_url [uint16] — 遷移先URL。
  • is_external [uint8] — 外部リンク(0/1)。
  • action_id [uint8] — アクション種別（遷移/送信/外部…）。
  • element_id [uint16] — 要素id。
  • element_class [uint16] — 要素class。
  • element_data [uint16] — data属性。

クリック座標（シグナル）

クリック位置(%)。ヒートマップ的な集計に。

  • page_x_pct [uint16] — クリックX位置(%)。
  • page_y_pct [uint16] — クリックY位置(%)。上部/下部の分析に。

gsc — 1行 = (ページ×検索語×日×検索面)

Search Console を (ページ×検索語×日) の粒度で保存。生PVログとJOIN可能。

tracking_id "all"（全サイト横断集計）: 非対応 — このサイト単体でのみ呼べる

キー & ID（キー）

allpv とは page_id が N:M（多対多）。無フィルタで JOIN すると行が掛け算で膨張（デカルト積＝行爆発）するため、先に url / keyword 等で絞ってから JOIN する。

  • page_id [uint32] [join] — ページ単位。allpv と N:M。無フィルタ JOIN は行爆発するので先に絞る。
  • query_id [uint32] — キーワードID。keyword で検索→query_id 集合。
  • search_type [uint8] — 1=web, 2=image, 3=video。

検索パフォーマンス（シグナル）

クリック・表示・CTR・掲載順位。position は列DBスキャンにリライトされ高速。

  • clicks [uint32] [✦] — 日次クリック数。
  • impressions [uint32] [✦] — 日次インプレッション数。
  • ctr [✦] — clicks / impressions。実行時算出（post_filter）。
  • position [✦] — 平均掲載順位。position_x100 に書き換わり列DBスキャン可・高速。
  • position_weighted — position × impressions。加重平均用。
  • position_x100 [uint16] — 順位×100 の生値。

解決カラム（シグナル）

query_id / page_id からラベル解決。

  • keyword — 検索語。query_id から解決。
  • url — ページURL。
  • title — ページタイトル。
  • page_type — ページタイプ。
  • page_fetch_status — 取得状態。

page_version — 1行 = ページの1つの版

ページ内容の版履歴。改稿の前後を混ぜずに比較できる。

tracking_id "all"（全サイト横断集計）: 非対応 — このサイト単体でのみ呼べる

キー & ID（キー）

allpv.version_id と結合。版は page × device 単位で管理する（PC と SP は別テンプレートで、改稿履歴が別系列）。このため device_id もキーの一部。

  • version_id [join] — PK / allpv.version_id との結合キー。
  • page_id [join] — ページ識別。allpv で URL 絞り→JOIN 推奨。
  • device_id — 1=PC, 2=SP, 3=tablet。PC/SP/tablet で版履歴が別系列なのでキーに含む。

版のメタ（シグナル）

版番号・作成日・URL。

  • version_no — 同一ページ×デバイス内の版番号。1が初版。calc MAX() で最新版数。
  • update_date — 版作成日。特定日の変更を探すのに。
  • url — URL。
  • title — タイトル。
  • device_type — PC/SP/tablet。

summary_allpage — 1行 = ページ×次元の期間累計

全ページのサマリーデータを返すマテリアル。ページ別集計に便利。

tracking_id "all"（全サイト横断集計）: 対応

グルーピング次元（キー）

集計の切り口。数値IDのまま（device_type は自動付与）。

  • page_id [uint32] — ページ識別。ランキングはこのマテリアル。
  • device_id [uint8] — 1=PC,2=SP,3=tablet。device_type 自動付与。
  • utm_source — 流入元（ラベル解決済み）。
  • utm_medium — メディア。
  • utm_campaign — キャンペーン。
  • source_domain — 流入元ドメイン。
  • is_newuser [uint8] — 新規(1)/リピート(0)。
  • is_QA [uint8] — QA経由フラグ。

累計メジャー（シグナル）

期間の累計値。数値範囲フィルタは post_filter。

  • pv_count [uint32] — 期間PV累計。
  • user_count [uint32] — 期間ユーザー累計。
  • bounce_count [uint32] — 期間直帰累計。
  • exit_count [uint32] — 期間離脱累計。
  • time_on_page [uint32] — 期間滞在時間合計(秒)。
  • lp_count [uint32] — 入口になった回数の累計。

解決カラム（シグナル）

page_id から復元。

  • title — タイトル。
  • url — URL。
  • wp_qa_id — WP投稿ID。
